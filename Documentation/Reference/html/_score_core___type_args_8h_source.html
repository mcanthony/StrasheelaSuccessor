14 /* // Musing on design
15 
16  In Strasheela the score object constructors are a large number of named arguments, and the constructor definition is spread over multiple classes. How can I implement something like that in C++?
17 
18  It is essential I get constructors with something like named arguments, even if they are not supported by bindings to other languages, because the complexity of creating score data gets unmanagable otherwise;
19  If necessary, I may have additional simpler constructors for bindings to other langs;
20 
21  - Specify a single nested data structure (e.g., vector) as argument, which contains all args, and which can be created in-place, e.g., the following (note: such constructor does not work with map, unfortunately). Is this too lengthy? Likely...
22  vector<string> args {"arg1", "x", "arg2", "y"};
23 
24  map<string,string> args2 {{"arg1", "x"}, {"arg2", "y"}};
25  for(auto x: args2)
26  std::cout << x.first << ": " << x.second << '\n';
27 
28  Lengthy type declaration can be reduced:
29  typedef map<string,string> args;
30  args myArgs {{"arg1", "x"}, {"arg2", "y"}};
31 
32  Constructor can be used in an expression, e.g., I can call an init method with
33  init(args {{"arg1", "x"}, {"arg2", "y"}})
34  init(arg1: "x", arg2: "y") // preferred syntax comparions -- much shorter, but the other is not unreasonable
35 
36  Problem: args are only captured at run-time, and so the compiler cannot help much.
37 
38  Arg values are boost::variant instances. boost::apply_visitor cares for compile-time error checking of types.
39 
40 
41  - Similar approach
42  http://www.reddit.com/r/programming/comments/19bul0/c11_named_parameters_using_operator_suffixes_and/
43 
44 
45 
46  - method chaining: http://www.parashift.com/c++-faq-lite/named-parameter-idiom.html
47 
48  - The Boost Parameter Library: http://www.boost.org/doc/libs/1_55_0/libs/parameter/doc/html/index.html
49 
50  - Variadic function:
51  https://en.wikipedia.org/wiki/Variadic_function#Example_in_C
52 
53  */
